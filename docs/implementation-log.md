# 実装履歴

## 2026-07-19 長期開発資料の整備

### 目的

会話の圧縮やセッション切替後も、地下百景の現状・設計意図・未対応事項を安全に引き継ぐ。

### 実装内容

`AGENTS.md` と `docs/` に、現在地、全体構成、ゲーム仕様、保存・ランキング仕様、規則、ADR、課題を追加した。READMEからも資料の入口を案内する。

### 主要な処理の流れ

次回の作業者は `AGENTS.md` → `README.md` → `docs/now.md` → 変更範囲の資料 → 関連コードの順で確認する。

### 設計上の注意点

資料はコードに代わる仕様ではない。通常は実行中コードを事実として扱い、保存・外部送信の不一致だけは実装前に確認する。

### 影響範囲

開発・保守手順のみ。ゲームの実行ロジック、セーブデータ、公開挙動は変更しない。

### 未対応・今後の課題

資料の更新を機能変更と同じコミットに保つ運用を継続する。

## 2026-07-19 オンラインランキング基盤

### 目的

プレイヤーが任意で到達記録を共有できるようにする。

### 実装内容

ゲームに地下ランキング画面と結果送信UIを追加し、`ranking-api/` にCloudflare Worker/D1のAPIと移行SQLを追加した。ゲームはGitHub Pagesのまま、ランキングだけ外部APIに分離した。

### 主要な処理の流れ

結果画面で公開名を入力して送信すると、匿名端末ID単位の自己ベストがWorkerへ送られる。WorkerはD1の総合・週間ベストを更新し、ランキング画面は上位100件を取得する。

### 設計上の注意点

テストモードと途中放棄は送信しない。送信は任意で、自動送信しない。クライアント値を信頼するカジュアルランキングである。

### 影響範囲

`app/page.tsx`、`app/ranking.ts`、`public/ranking-config.js`、`ranking-api/`、Cloudflare D1/Worker、GitHub Pages。

### 未対応・今後の課題

不正対策を強化するなら、サーバー発行シードと行動ログ検証を設計する。
