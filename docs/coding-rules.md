# コーディング規則

最終確認: 2026-07-19

- TypeScriptの既存スタイルに合わせる。ゲーム状態は `Run`、恒久進行は `Meta` を中心に扱う。
- `Run`を変更する機能は、開始、復元、保存、終了、テストモードで必要な補完を確認する。
- `Meta`を変更する機能は、旧データの正規化と既存値の保持を確認する。
- マスター追加は可能な限り `app/monsters.ts`、`app/items.ts` のデータ配列へ置く。
- UIはスマートフォン表示を先に確認し、タップ・キーボードの両方を既存仕様どおりに保つ。
- 外部通信は目的、送信項目、送信契機を明示する。自動送信を追加しない。
- 画像・音声を大量に追加する前に、容量と初回転送量を確認する。既存のCSS/Web Audio方針を優先する。
- 変更後は `npm test`、公開に関わる変更では `npm run build:pages` を実行する。Lintが必要な変更では `npm run lint` も実行する。
