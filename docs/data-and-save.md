# データ・セーブ仕様

最終確認: 2026-07-22

## 端末内データ

ブラウザの `localStorage` を使う。通常プレイは自動保存される。

| キー | 内容 | 互換性上の注意 |
| --- | --- | --- |
| `chika-hyakkei-run-v3` | 現在の冒険。階層、地図、装備、戦闘、状態異常、商店、遺物など | 冒険中のみ存在する |
| `chika-hyakkei-run-backup-v1` | 現在セーブを書き換える直前の正常な冒険 | 正常終了時は現在セーブと一緒に削除する |
| `chika-hyakkei-run-quarantine-v1` | 読み込めなかった現在セーブの原文 | 自動削除せず、調査・手動復旧用に保持する |
| `chika-hyakkei-meta-v2` | 最高記録、図鑑、熟練度、称号、墓、職業解放 | `v1`から不足値を補完して読む |
| `chika-hyakkei-meta-backup-v1` | Metaを書き換える直前の正常値 | 図鑑・熟練度など恒久記録の1世代バックアップ |
| `chika-hyakkei-meta-quarantine-v1` | 読み込めなかったMetaの原文 | 自動削除せず、調査・手動復旧用に保持する |
| `chika-hyakkei-test-record-v1` | 端末限定の匿名テスト記録 | 自動送信しない |
| `chika-hyakkei-ranking-player-v1` | ランキング自己ベストを識別する匿名ID | 公開名・メールアドレスではない |
| `chika-hyakkei-ranking-pending-v1` | 送信に失敗した終了記録（最大20件） | 接続可能な次回起動時に同じIDで再送する |

現在RunとMetaは保存前に構造を検査し、直前の正常値をバックアップする。現在値がJSON破損または必須項目不足なら原文を隔離し、正常なバックアップがあれば自動復元する。描画中の例外では、再読込・直前Run・現在Runを隔離してタイトルへ戻る、の選択肢を表示する。

## 変更時のルール

保存形式を変える前に、変更前後・既存データへの影響・補完/移行・ロールバック方法を `implementation-log.md` に記録する。既存値はデフォルト補完を優先し、明示的な許可なく削除しない。

## 匿名テスト記録

テスト記録は端末内にだけ残る。含むのはランダムな端末ID、利用日、プレイ/到達/結果の集計、直近結果であり、名前・メール・位置情報は含まない。

## 匿名アクセス集計

Cloudflare Web Analyticsをページ表示時に読み込む。これはゲーム内の端末記録・セーブ・ランキングとは別で、ページビュー、匿名化された訪問集計、参照元、端末・表示性能の傾向をCloudflareダッシュボードで確認するために使う。ゲーム内の名前、プレイ内容、localStorageの値は送信しない。ブラウザの追跡防止設定などにより集計は実数より少なくなることがある。

## オンラインランキング

- 通常プレイの死亡、中断、帰還、踏破時に自動送信する。テストモードは送信しない。
- 終了記録には冒険終了ごとの `submissionId` を付ける。送信前に端末キューへ保存し、成功後だけキューから除く。
- WorkerとD1は `submissionId` を一意に扱い、通信結果を受け取れず再送した場合も同じ記録を二重登録しない。
- 公開するのは公開名、職業、階層、得点、討伐数、階層主数、結果、日時。
- D1には終了した冒険ごとの記録を保存する。
- 現段階ではクライアント申告値を検証しないカジュアルランキング。厳密な順位保証はしない。
- タイトル画面では、公開名・到達階だけの総合上位3件を自動取得して表示する。

終了後は総合ランキングを自動で開く。送信された今回の行は、Workerが返す記録IDを使って黄色に点滅表示する。週間ランキングは利用者が増えるまで画面・APIとも休止するが、将来の再開に備えてD1の `week_key` は保持する。

WorkerのAPI、テーブル、移行手順は `ranking-api/` を参照する。
