# データ・セーブ仕様

最終確認: 2026-07-19

## 端末内データ

ブラウザの `localStorage` を使う。通常プレイは自動保存される。

| キー | 内容 | 互換性上の注意 |
| --- | --- | --- |
| `chika-hyakkei-run-v3` | 現在の冒険。階層、地図、装備、戦闘、状態異常、商店、遺物など | 冒険中のみ存在する |
| `chika-hyakkei-meta-v2` | 最高記録、図鑑、熟練度、称号、墓、職業解放 | `v1`から不足値を補完して読む |
| `chika-hyakkei-test-record-v1` | 端末限定の匿名テスト記録 | 自動送信しない |
| `chika-hyakkei-ranking-player-v1` | ランキング自己ベストを識別する匿名ID | 公開名・メールアドレスではない |

## 変更時のルール

保存形式を変える前に、変更前後・既存データへの影響・補完/移行・ロールバック方法を `implementation-log.md` に記録する。既存値はデフォルト補完を優先し、明示的な許可なく削除しない。

## 匿名テスト記録

テスト記録は端末内にだけ残る。含むのはランダムな端末ID、利用日、プレイ/到達/結果の集計、直近結果であり、名前・メール・位置情報は含まない。

## オンラインランキング

- 送信は結果画面でプレイヤーが押した時だけ。
- 公開するのは公開名、職業、階層、得点、討伐数、階層主数、結果、日時。
- テストモード・途中放棄は送信しない。
- D1には端末ごとに総合・週間それぞれの自己ベストを保存する。
- 現段階ではクライアント申告値を検証しないカジュアルランキング。厳密な順位保証はしない。

WorkerのAPI、テーブル、移行手順は `ranking-api/` を参照する。
